## Working Preferences

- We communicate in English during Codex sessions.
- All project-facing content must be written in Catalan.
- This includes code comments, UI text, documentation, commit-related project text, and functional descriptions unless there is a clear project reason to use another language.
- Avoid wording that looks obviously AI-generated in any project artifact.
- Keep style natural, consistent with the repository, and aligned with the existing project tone.
- Backend is the main work area for these sessions.
- Keep the existing frontend structure and styles whenever possible.
- Only change frontend code when it is strictly necessary to connect or validate backend functionality.
- Any frontend change should be minimal to reduce conflicts with other collaborators.
